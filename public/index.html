REST API for Elden Ring Community Website

üêâ Bosses Endpoints

GET /api/bosses

Get all bosses with optional pagination and search

Query Parameters (Optional):

  • limit integer - Number of items to return (default: 10, max: 100)
  • offset integer - Number of items to skip (default: 0)
  • search string - Search by boss name or location
Example Request: 
GET http://localhost:3000/api/bosses?limit=5&offset=0&search=malenia

Example Response:

{
  "data": [
    {
      "id": 1,
      "name": "Malenia, Blade of Miquella",
      "title": "Goddess of Rot",
      "description": "...",
      "location": "Elphael, Brace of the Haligtree",
      "difficulty": 5,
      "health": 33251,
      "drops": "Remembrance of the Rot Goddess",
      "order": 0,
      "image": null
    }
  ],
  "pagination": {
    "total": 6,
    "limit": 5,
    "offset": 0,
    "hasMore": true
  }
}

POST /api/bosses

Create a new boss

Request Body:

  • name string *required - Boss name (2-255 characters, letters only)
  • title string - Boss title (max 255 characters)
  • description string *required - Description (min 10 characters)
  • location string - Boss location (max 255 characters)
  • difficulty integer *required - Difficulty level (1-5)
  • health integer - Boss health points
  • drops string - Item drops (max 255 characters)
  • order integer - Display order (default: 0)
Example Request: 
POST http://localhost:3000/api/bosses
Content-Type: application/json

{
  "name": "Radahn",
  "title": "Starscourge",
  "description": "A powerful demigod...",
  "location": "Redmane Castle",
  "difficulty": 5,
  "health": 9572,
  "drops": "Remembrance of the Starscourge",
  "order": 4
}

Same fields as POST endpoint

  "title": "Goddess of Rot - Updated",

  "location": "Elphael, Brace of the Haligtree",
  "difficulty": 5,
  "health": 33251,
  "drops": "Remembrance of the Rot Goddess",
  "order": 1
}
DELETE /api/bosses/:id

Delete a boss

Example Request: 
DELETE http://localhost:3000/api/bosses/1

Example Response:

{
  "message": "Boss deleted successfully"
}

üì∞ News Endpoints

GET /api/news

Get all news with optional pagination and search

Query Parameters (Optional):

  • limit integer - Number of items to return (default: 10, max: 100)
  • offset integer - Number of items to skip (default: 0)
  • search string - Search by title or content
Example Request: 
GET http://localhost:3000/api/news?limit=5&offset=0
GET /api/news/:id

Get a single news article by ID

Example Request: 
GET http://localhost:3000/api/news/1
POST /api/news

Create a new news article

Request Body:

  • title string *required - Article title (5-255 characters)
  • content string *required - Article content (min 20 characters)
  • user_id integer *required - ID of the user creating the article
  • publication_date string (ISO 8601) - Publication date (default: current date)
Example Request: 
POST http://localhost:3000/api/news
Content-Type: application/json

{
  "title": "New DLC Announced",
  "content": "FromSoftware has announced...",
  "user_id": 1,
  "publication_date": "2026-01-04T00:00:00.000Z"
}
PUT /api/news/:id

Update an existing news article

Request Body:

Same fields as POST endpoint (except user_id cannot be changed)

DELETE /api/news/:id

Delete a news article

Example Request: 
DELETE http://localhost:3000/api/news/1
